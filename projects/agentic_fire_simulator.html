Agentic Fire Simulator

Built a grid-based firefighting simulation using Python and Pygame, integrating reinforcement learning with cooperative agents (water and shovel) to control spreading wildfires. Designed a shared Q-table system enabling agents to learn collaboratively, implemented real-time visualization, and tracked performance metrics across episodes.

Technologies Used

  • Python
  • NumPy
  • Pygame
  • Q-learning
  • Multi-Agent System

Pygame Simulation

Cooperative RL Fire Simulation — Project Rundown

Objective

  • Simulate wildfire spread and management using cooperative reinforcement learning with multiple agents.

Core Components

  • Environment (model.py) — Grid world with trees, fire, burned areas, and trenches; handles fire spread and rules.
  • Agents (agents.py)
    • FirefighterWater: extinguishes fires.
    • FireFighterShovel: digs trenches to block fire spread.
  • Q-learning System — Agents share a Q-table to learn collaboratively.

Code Flow

  • Initialization (create_simulation)
    • Builds environment with adjustable tree density.
    • Sets up a shared Q-table (fresh or loaded from prior runs).
    • Spawns three agents at strategic starting positions.
  • Main Loop (main)
    • Handles user input (quit/reset).
    • Advances environment steps (fire spreads, agents act).
    • Renders grid, agents, and stats in real time via Pygame.
    • Resets episode if the fire engulfs the map, carrying over the learned Q-table.

Visualization & Debugging

  • Color-coded grid for fire, trees, trenches, and agents; shovel agent has a highlighted digging radius.
  • On-screen stats: episode, active fires, trees remaining, trenches dug, extinguished fires, Q-table size.
  • Optional console debug: local grid snippets near agents.

Key Outcomes

  • Demonstrates multi-agent cooperation to control wildfire spread.
  • Provides a visual, interactive simulation of reinforcement learning.
  • Tracks performance metrics across episodes to monitor learning progress.
